# v4.0.0 Release Notes (Draft)

## Stable Release Preparation

Version 4.0.0 will be the first stable release of IBIS Auto Downloader. This
draft is release-note material only; it does not create a GitHub Release.

## Highlights

- Reliable invoice discovery across DevExpress grid pages and billing periods.
- Download completion checks that preserve the server-provided Excel extension
  and avoid accepting temporary browser files.
- Resume verification that requeues missing outputs while keeping completed
  invoices distinct by invoice ID and billing period.
- Structured retry policy and browser/session AutoRecovery with manual IBIS
  login.
- Billing-period selection, read-only dry-run previews, and configurable
  download/report directories.
- JSON, CSV, and HTML run reports plus performance and diagnostic summaries.

## Upgrade Notes

- Existing completed-invoice and download-state files remain supported.
- Existing default download, report, log, and state locations remain unchanged.
- The application continues to require manual login in Chrome and does not
  persist IBIS credentials.

## Operational Guidance

- Use `python3 main.py --show-config` before deployment to verify runtime
  configuration without starting a browser.
- Use `python3 main.py --dry-run` to inspect the selected queue safely.
- Retain `logs/`, `state/`, and generated reports when diagnosing an issue.

## Release Validation Checklist

- Run the complete automated test suite.
- Validate normal download, dry-run, resume, reporting, and recovery against
  the intended IBIS environment.
- Confirm that generated state and reports remain internally consistent.
- Review the release tag, changelog, and user documentation before publishing.
